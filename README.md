# 基于ESP32的日式女仆风格对话与灯控系统
## 项目概述
ESP32开发板上的MicroPython应用，核心为**日式女仆语气智能对话**与**WS2812灯带精准控制**，新增SD卡操作库预留扩展能力，可支撑后续麦克风录音、扬声器音频、显示屏素材、网页配网配置等文件存储需求，适配智能家居趣味交互与功能扩展场景。


## 核心功能
| 功能模块         | 具体描述                                                                 |
|------------------|--------------------------------------------------------------------------|
| 日式女仆对话     | 对接DeepSeek大模型，以“哦/呢”句尾语气交互，支持日常聊天、补气中药知识讲解 |
| WS2812灯控       | 3路灯珠独立控制（亮灭/颜色调节），自然语言触发（如“第2个灯亮淡黄色”）     |
| 指令管理         | 3个核心指令：`show`查聊天记录摘要、`clear`清空历史、`exit`保存记录退出   |
| 聊天记录管理     | 自动生成对话摘要，存储于ESP32本地Flash（`chat_summary.txt`），超6000字符自动截断防内存溢出 |
| WiFi自动连接     | 最多3次重试连接，成功显示IP，断网提示状态                                 |
| SD卡扩展支持（新增） | 集成SD卡操作库，支持文件读写、目录管理，为后续麦克风录音、显示屏素材、网页配网配置等功能预留存储能力 |


## 硬件需求
- ESP32开发板（如ESP32-WROOM-32）
- WS2812灯带/灯珠（3个，支持单灯独立控制）
- SD卡模块+SD卡（可选，用于后续功能扩展存储）
- 杜邦线、USB数据线（供电/烧录）


## 硬件接线
| 模块       | ESP32引脚 | 说明                          |
|------------|-----------|-------------------------------|
| WS2812     | IO18      | 数据信号输入（控制颜色/亮灭） |
| WS2812     | 3.3V      | 电源（勿接5V，避免烧毁）      |
| WS2812     | GND       | 接地（防信号干扰）            |
| SD卡模块（扩展） | IO6(SCK)  | SPI时钟引脚（后续功能使用）   |
| SD卡模块（扩展） | IO4(CS)   | 片选引脚（后续功能使用）      |
| SD卡模块（扩展） | IO7(MISO) | 数据输入引脚（后续功能使用）  |
| SD卡模块（扩展） | IO5(MOSI) | 数据输出引脚（后续功能使用）  |
| SD卡模块（扩展） | 3.3V/GND  | 电源/接地（后续功能使用）     |


## 软件环境与依赖
### 1. 基础环境
- 开发工具：Thonny（代码编辑/烧录/交互）
- 固件：MicroPython for ESP32（v1.20+）
- 联网：ESP32需连WiFi（下载依赖/对接大模型API）

### 2. 依赖库安装
ESP32联网后，在Thonny终端执行：
```python
import mip
# 下载HTTP请求库（对接DeepSeek大模型）
mip.install("urequests")
```


## 代码文件说明
| 文件名               | 作用                          | 核心关联                     |
|----------------------|-------------------------------|------------------------------|
| `main.py`            | 程序入口                      | 调度WiFi/对话/灯控逻辑        |
| `config.py`          | 配置文件                      | WiFi/API密钥/WS2812参数/女仆语气提示词 |
| `wifi_manager.py`    | WiFi管理                      | 连接/重试/IP获取             |
| `deepseek_client.py` | 大模型对接                    | 流式对话/聊天历史管理        |
| `ws2812_controller.py` | 灯控核心                    | 单灯/全灯颜色控制            |
| `light_parser.py`    | 灯控指令解析                  | 提取大模型回复中的灯控指令   |
| `cmd_handler.py`     | 指令处理                      | 处理`show`/`clear`/`exit`    |
| `sdcard.py`          | SD卡操作库（新增）            | 硬件初始化/文件读写/目录管理，为后续扩展功能预留 |
| `chat_summary.txt`   | 聊天记录文件                  | 存储于ESP32 Flash，`exit`时自动保存 |


## 使用步骤
1. **硬件接线**：连接ESP32与WS2812；若需预留SD卡扩展，按“硬件接线”连接SD卡模块；
2. **代码烧录**：通过Thonny将所有`.py`文件烧录到ESP32；
3. **配置修改**：打开`config.py`，替换WiFi账号密码、DeepSeek API密钥；
4. **启动程序**：运行`main.py`，ESP32自动连接WiFi；
5. **交互操作**：
   - 聊天：输入问题（如“讲个黄芪的故事”）；
   - 灯控：输入需求（如“第2个灯亮淡黄色”）；
   - 指令：`show`查记录、`clear`清空历史、`exit`保存记录并退出。


## 注意事项
1. DeepSeek API密钥：需在官网申请，替换`config.py`中`API_KEY`，否则无法对接大模型；
2. 聊天记录存储：`chat_summary.txt`默认存于ESP32 Flash，非SD卡；
3. SD卡扩展：当前仅集成SD卡库，后续需开发麦克风/显示屏功能时，直接调用`sdcard.py`接口即可；
4. 灯珠索引：“第1个灯”对应代码`index:0`，“第2个”对应`index:1`，“第3个”对应`index:2`；
5. 内存限制：对话摘要超6000字符自动截断，避免ESP32内存溢出。
