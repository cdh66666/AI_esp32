# 基于ESP32的日式女仆风格对话与灯控系统
## 项目概述
ESP32开发板上的MicroPython应用，核心为**日式女仆语气智能对话**与**WS2812灯带精准控制**，集成NS4168音频模块支持测试音播放、XL9555 IO扩展芯片驱动，新增SD卡操作库预留扩展能力，可支撑后续麦克风录音、扬声器音频、显示屏素材、网页配网配置等文件存储需求，适配智能家居趣味交互与功能扩展场景。

## 核心功能
| 功能模块         | 具体描述                                                                 |
|------------------|--------------------------------------------------------------------------|
| 日式女仆对话     | 对接DeepSeek大模型，以“哦/呢”句尾语气交互，支持日常聊天、补气中药知识讲解 |
| WS2812灯控       | 3路灯珠独立控制（亮灭/颜色调节），自然语言触发（如“第2个灯亮淡黄色”）     |
| 音频输出（新增） | 集成NS4168音频模块，支持固定幅度测试音、1kHz正弦波播放，可扩展语音交互功能 |
| IO扩展（新增）   | 通过XL9555芯片实现IO口扩展，支持多按键检测、多设备控制，提升硬件扩展能力 |
| 指令管理         | 3个核心指令：`show`查聊天记录摘要、`clear`清空历史、`exit`保存记录退出   |
| 聊天记录管理     | 自动生成对话摘要，存储于ESP32本地Flash（`chat_summary.txt`），超6000字符自动截断防内存溢出 |
| WiFi自动连接     | 最多3次重试连接，成功显示IP，断网提示状态，保障大模型对接稳定性 |
| SD卡扩展支持     | 集成SD卡操作库，支持文件读写、目录管理，为后续麦克风录音、显示屏素材、网页配网配置等功能预留存储能力 |

## 硬件需求
- ESP32开发板（如ESP32-WROOM-32）
- WS2812灯带/灯珠（3个，支持单灯独立控制）
- NS4168音频模块（可选，用于音频输出功能）
- XL9555 IO扩展芯片（可选，用于IO口扩展）
- SD卡模块+SD卡（可选，用于后续功能扩展存储）
- 按键（可选，配合XL9555实现按键控制）
- 杜邦线、USB数据线（供电/烧录）

## 软件环境与依赖
### 1. 基础环境
- 开发工具：Thonny（代码编辑/烧录/交互）
- 固件：MicroPython for ESP32（v1.20+）
- 联网：ESP32需连WiFi（下载依赖/对接大模型API）
- 依赖库目录：项目根目录下需包含`lib`文件夹，存放第三方依赖库

### 2. 依赖库安装
ESP32联网后，在Thonny终端执行以下命令安装核心依赖库，其他依赖库可放入`lib`文件夹直接调用：
```python
import mip
# 下载HTTP请求库（对接DeepSeek大模型）
mip.install("urequests")
```

### 3. `lib`文件夹说明
`lib`文件夹用于存放项目所需第三方依赖库，无需手动安装，ESP32运行时会自动识别调用，常见包含内容如下（根据实际使用场景调整）：
- 硬件驱动库：如特定型号音频芯片、传感器的MicroPython驱动
- 功能扩展库：如JSON解析增强库、加密库等
- 工具类库：如日志管理、数据格式转换相关库

## 使用步骤
1. **硬件接线**：
   - 基础功能：连接ESP32与WS2812（按“硬件接线”表对应引脚）；
   - 扩展功能：如需音频输出，连接NS4168模块；如需多IO控制，连接XL9555及按键；如需存储扩展，连接SD卡模块；
2. **代码烧录**：
   - 通过Thonny将所有`.py`文件及`lib`文件夹（含依赖库）烧录到ESP32；
   - 确保`chat_summary.txt`在首次运行后自动生成（无需手动创建）；
3. **配置修改**：
   - 打开`config.py`，替换`WIFI_SSID`和`WIFI_PASSWORD`为实际WiFi账号密码；
   - 替换`API_KEY`为个人申请的DeepSeek API密钥（官网申请，确保密钥有效）；
   - 按需调整`LED_PIN`（WS2812引脚）、`LED_COUNT`（灯珠数量）等硬件参数；
4. **启动程序**：
   - 在Thonny中运行`main.py`，ESP32会自动尝试连接WiFi，成功后显示IP地址；
   - 若连接失败，根据终端提示检查WiFi信息或硬件接线；
5. **交互操作**：
   - 日常聊天：输入任意话题（如“讲个黄芪的故事”“今天天气怎么样呢”），助手以女仆语气回复；
   - 灯控操作：输入自然语言指令（如“第2个灯亮淡黄色”“所有灯灭”），系统自动解析并控制WS2812；
   - 音频测试：在代码中调用`ns4168_audio.py`的`play_test_tone()`或`play_sine()`方法，测试音频输出；
   - 按键控制：若连接XL9555和按键，运行`xl9555.py`可检测按键状态（按下显示“●”，松开显示“○”）；
   - 指令使用：
     - 输入`show`：查看当前聊天记录摘要；
     - 输入`clear`：清空历史聊天记录（本地文件与内存数据均重置）；
     - 输入`exit`：保存当前聊天摘要到`chat_summary.txt`，并退出程序。

## 注意事项
1. **DeepSeek API密钥**：需在DeepSeek官网申请个人API密钥，替换`config.py`中`API_KEY`，否则无法对接大模型；密钥需注意保密，避免泄露导致额外费用；
2. **硬件供电**：
   - WS2812仅支持3.3V供电，勿接5V，否则会烧毁灯珠；
   - 扩展模块（NS4168、XL9555）需确保供电稳定，建议使用独立电源模块（如5V/2A）供电，避免ESP32主板供电不足；
3. **内存限制**：ESP32内存有限，对话摘要超过6000字符会自动截断，避免内存溢出导致程序崩溃；
4. **SD卡使用**：当前`sdcard.py`仅集成基础功能，后续开发扩展功能（如录音、存图）时，需调用`mount()`方法挂载SD卡后再进行文件操作；
5. **灯珠索引规则**：“第1个灯”对应代码中的`index:0`，“第2个灯”对应`index:1`，“第3个灯”对应`index:2`，输入指令时需注意顺序匹配；
6. **调试功能**：如需开启ESP32调试或WebREPL功能，可取消`boot.py`中对应代码的注释（`import esp`/`webrepl.start()`），调试完成后建议重新注释，避免占用资源；
7. **中文输入**：`chinese_input.py`已适配中文输入，若出现输入乱码，检查Thonny终端编码是否为UTF-8（设置路径：Thonny→工具→选项→终端→编码→UTF-8）。
